# Tech Review Methodology Skill

## 리뷰 프레임워크

### Phase 1: 비즈니스 해체
```
자연어 요구사항
       ↓
┌─────────────────────────────────────┐
│  Business Decomposition             │
│  - 도메인 식별 (B2B/B2C/B2B2C)      │
│  - 이해관계자 매핑                   │
│  - 컴플라이언스 도출                 │
│  - 워크플로우 정의                   │
└─────────────────────────────────────┘
```

### Phase 2: 아키텍처 순서 검토
- [ ] Test 설계가 구현보다 선행하는가?
- [ ] Contract-First 접근인가?
- [ ] 비즈니스 규칙이 실행 가능한 명세로 정의되었는가?

### Phase 3: 종속성 분석
| 검토 항목 | 위험 신호 |
|----------|----------|
| 라이선스 | BSL, SSPL 전환 이력 |
| 표준 준수 | 특정 도구 DSL만 지원 |
| 추상화 | 단일 구현체 직접 의존 |

### Phase 4: 확장성 평가
토큰 효율화 대안 검토:
1. **Symbolic Index** - 벡터 검색 대신 명시적 참조
2. **프롬프트 캐싱** - 반복 시스템 프롬프트 재사용
3. **Local OLAP** - DuckDB로 구조화된 메타데이터 쿼리
4. **하이브리드** - 쿼리 유형별 최적 전략 선택
