# Web Design Guide

Reference documentation for web accessibility, responsive design, and UI best practices with 100+ actionable rules.

## Overview

Covers WCAG 2.1 compliance (Level AA standard), POUR principles (Perceivable, Operable, Understandable, Robust), color contrast ratios, keyboard navigation, ARIA landmarks and attributes, semantic HTML, responsive design patterns, and performance-conscious CSS. Referenced by `fe-design-expert` and the `web-design-guidelines` skill for UI code review.

## Key Topics

- WCAG 2.1 compliance levels and POUR principles
- Color contrast requirements: 4.5:1 normal text, 3:1 large text/UI components
- Keyboard navigation patterns and focus management
- ARIA landmarks, roles, and attributes (`aria-label`, `aria-live`, `aria-expanded`)
- Semantic HTML structure (`<header>`, `<nav>`, `<main>`, `<aside>`, `<footer>`)
- Responsive design with CSS Grid and Flexbox
- Testing tools for accessibility validation
