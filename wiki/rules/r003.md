# R003: Interaction Rules

Defines response style, status format, and request handling behavior for all agent responses.

## Overview

Ensures consistent, actionable communication. Responses should be brief and specific. Output verbosity adapts based on task complexity and active modes (ecomode, effort level). Status reporting uses a standard format so users can track progress uniformly.

## Priority

**SHOULD** — Strong recommendation. Violations reduce communication quality but do not block operations.

## Key Requirements

**Status format:**
```
[Start] {task name}
[Progress] {current step} ({n}/{total})
[Done] {task name} — Result: {summary}
[Failed] {task name} — Cause: {reason} — Alternative: {solutions}
```

**Request handling:**

| Type | Action |
|------|--------|
| Clear | Execute immediately |
| Ambiguous | `[Confirm] Understood "{request}" as {interpretation}. Proceed?` |
| Risky | `[Warning] This action has {risk}. Continue? Yes: {action} / No: Cancel` |

**Output style selection priority:**
1. User explicit request → override
2. Ecomode active ([[r013]]) → force `concise`
3. Agent effort level → map to style (`low`=concise, `medium`=balanced, `high`=explanatory)
4. Default → `balanced`

## Session-Level Style Enforcement

Per-response style selection (concise/balanced/explanatory) is R003's responsibility. Session-wide default tone and format are handled by the Output Styles layer — a separate CC-native mechanism.

| Layer | Responsibility | Trigger |
|-------|---------------|---------|
| R003 (this rule) | Per-response style criteria | Prompt-based, advisory |
| R013 ([[r013]]) | Ecomode concise override | Dynamic, context-triggered |
| Output Styles | Session-wide tone/format | Static, session-level |

Default active style: `korean-engineer` (`.claude/output-styles/korean-engineer.md`). See [output-styles guide](../guides/output-styles.md).

## Enforcement

Prompt-based (CLAUDE.md + rules/) per [[r021]]. Advisory only.
