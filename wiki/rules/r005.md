# R005: Optimization Guide

Provides guidelines for efficiency, token optimization, and task batching — applied selectively where benefit is clear and measurable.

## Overview

Optimization should be applied to repetitive or bottlenecked operations. Strategies include parallelization, caching, lazy loading, and token compression. Readability always takes precedence — no optimization without measurable benefit.

## Priority

**MAY** — Optional. Apply when there is a clear, measurable gain. Never at the cost of readability or correctness.

## Key Requirements

**Efficiency strategies:**

| Strategy | When | Example |
|----------|------|---------|
| Parallel | 3+ independent I/O tasks | Read multiple files simultaneously |
| Caching | Same data accessed repeatedly | Cache file contents, reuse search results |
| Lazy Loading | Large datasets, partial use | Read only needed files, stream results |

**Token optimization:**
- Include only necessary info, remove duplicates, use summaries
- Concise expressions, minimize code blocks, no unnecessary repetition

**Task optimization:**
- **Batch**: Group similar tasks (edit 10 files at once)
- **Incremental**: Process only changed parts

**When to optimize:**

| Do | Don't |
|----|-------|
| Repetitive tasks, clear bottleneck, measurable gain | One-time tasks, already fast, complexity > benefit |

**Context optimization via HTML comments (v2.1.72+):** Move model-unnecessary metadata into HTML comments in `.md` files to reduce context token usage.

## Enforcement

**MAY** — No enforcement mechanism. Applied at agent discretion.
