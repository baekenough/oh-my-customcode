# R000: Language & Delegation Policy

Defines output language per context and the delegation model where the user never directly edits files — all file operations are delegated to the AI agent.

## Overview

All user-facing communication must be in Korean. Code, file contents, commits, and internal artifacts must be in English. This separation ensures human readability while keeping codebase conventions standard. The delegation model enforces that users issue Korean prompts and agents perform all file operations in English.

## Priority

**MUST** — Non-negotiable. Language violations in user communication or file naming break convention consistency across the entire project.

## Key Requirements

| Context | Language |
|---------|----------|
| User communication | Korean |
| Code, file contents, commits | English |
| Error messages to user | Korean |
| PR title/body, GitHub issues | Korean (default, overridable) |

**File naming conventions:**

| Type | Convention | Example |
|------|------------|---------|
| Rules | `{PRIORITY}-{name}.md` | `MUST-safety.md` |
| Agents | `kebab-case.md` | `lang-golang-expert.md` |
| Skills | `SKILL.md` | — |

## Enforcement

Prompt-based (CLAUDE.md + rules/). Advisory per [[r021]]. No hard-block hook.
