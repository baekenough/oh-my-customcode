# R010: Orchestrator Coordination Rules

The main conversation is the sole orchestrator and MUST NEVER directly write, edit, or create files — all file modifications MUST be delegated to appropriate subagents.

## Overview

The orchestrator's role is routing and coordination only. File operations, git commands, and all write actions must go through specialist subagents. This ensures separation of concerns, maintains audit trails, and prevents accidental bulk changes from the orchestrator context.

## Priority

**MUST** — Critical. Direct file writes from the orchestrator are violations. Advisory PostToolUse hook monitors compliance. Candidate for hard-block promotion per [[r021]].

## Key Requirements

**Delegation table (key entries):**

| Task Type | Required Agent |
|-----------|---------------|
| Create agent/skill/guide | `mgr-creator` |
| Git operations | `mgr-gitnerd` |
| Memory operations | `sys-memory-keeper` |
| Python/FastAPI | `lang-python-expert` / `be-fastapi-expert` |
| Go code | `lang-golang-expert` |
| TypeScript/Next.js | `lang-typescript-expert` / `fe-vercel-agent` |
| Docker/Infra | `infra-docker-expert` |
| Database schema | `db-supabase-expert` |
| Unmatched specialized task | `mgr-creator` → dynamic agent creation |

**Protected paths (mgr-creator required):**
- `.claude/agents/*.md`
- `.claude/skills/*/SKILL.md`
- `guides/*/` (new directories)

**Orchestrator allowed operations:** Read, Glob, Grep only (read-only).

**Dynamic agent creation:** When no matching agent exists → delegate to `mgr-creator` with domain keywords, file patterns, and required capabilities. mgr-creator auto-discovers relevant skills/guides and creates the agent.

**Agent Teams exception:** Agent Teams members are peers and CAN spawn sub-agents to execute complex workflows.

## Agent Capability Pre-Check (v0.147.0)

Before delegating a task to a subagent, MUST verify the target agent's tool capabilities against the task requirements. Failure to pre-check causes round-trip waste (delegation → failure → re-delegation).

| Task involves | Verify in target agent frontmatter |
|--------------|-----------------------------------|
| `gh` / shell commands | `tools:` includes Bash AND `disallowedTools:` excludes Bash |
| `Read` external files | `tools:` includes Read |
| `Write` files | `tools:` includes Write |
| MCP server calls | `mcpServers:` includes the required server |

**Known limitation:** `arch-documenter` has `disallowedTools: [Bash]` — cannot run `gh` or shell scripts. Pre-collect data via orchestrator and pass as content, or use `general-purpose` for the Bash-needing portion.

Reference issues: #1202 item #2, `feedback_arch_documenter_no_bash.md`.

## Sensitive Path Handling

> **Status (CC v2.1.121+):** DEPRECATED. Direct Write/Edit/Bash on `.claude/`, `.git/`, `.vscode/` works without prompts under `mode: "bypassPermissions"`. The `/tmp/*.sh` bypass pattern previously required is no longer necessary. See #1101.

The legacy `/tmp/*.sh` script bypass pattern applied to CC < v2.1.121 only. For current CC versions, `mode: "bypassPermissions"` on every Agent tool call (see Universal bypassPermissions) is sufficient.

## Enforcement

Advisory PostToolUse hook (`git-delegation-guard.sh`) + PostCompact re-injection per [[r021]]. Candidate for hard-block if direct-write violations exceed 3/session.
