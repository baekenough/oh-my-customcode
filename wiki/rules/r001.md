# R001: Safety Rules

Defines absolutely prohibited actions and required verification steps before any destructive operation.

## Overview

Prevents agents from performing irreversible or dangerous actions such as exposing secrets, deleting system files, or making unauthorized network requests. Any potential violation triggers an immediate full stop and user notification.

## Priority

**MUST** — Non-negotiable. Safety violations can cause irreversible data loss or security breaches.

## Key Requirements

**Prohibited actions:**

| Category | Prohibited |
|----------|-----------|
| Data | Expose API keys/secrets/passwords, collect PII, log auth tokens |
| File System | Modify `/etc`, `/usr`, `/bin`; delete outside project; modify `.env`/`.git/config` without approval |
| Commands | `rm -rf /` or broad deletes, shutdown/restart, sudo/su, network config changes |
| External | Access URLs without approval, send user data externally, download/execute unknown scripts |

**Before any destructive operation:**
1. Verify target
2. Assess impact scope
3. Check recoverability
4. Get user approval

**On violation:**
1. Stop all operations
2. Preserve current state
3. Report: what detected, why risky, action taken
4. Wait for instructions

## Destructive Git Commands (v0.136.1, #1146)

Added in v0.136.1 after the v0.136.0 working tree loss incident. The following git commands require explicit user approval per invocation:

| Command | Risk | Required Action |
|---------|------|----------------|
| `git reset --hard <ref>` | Erases uncommitted + committed local changes | Confirm `git status` clean; show ref delta; explicit approval |
| `git checkout -- <path>` / `git restore <path>` | Discards uncommitted file changes | Confirm intentional revert; explicit approval |
| `git clean -fd` / `git clean -fdx` | Permanently deletes untracked files | List with `git clean -nd` first; explicit approval |
| `git branch -D <name>` (unmerged commits) | Loses unmerged work | Show `git log <branch>` first; confirm pushed; explicit approval |
| `git push --force` / `--force-with-lease` to shared branches | Rewrites shared history | NEVER on main/master; explicit approval for feature branches |

Recovery hint: `git reflog` — most operations recoverable within 30 days.

See [[git-safety]] for full pre-flight checks, recovery procedures, and agent-specific rules.

## Enforcement

Prompt-based (CLAUDE.md + rules/) per [[r021]]. Some specific patterns enforced by `stage-blocker` hard-block hook. Destructive git command rules are advisory (prompt-based); a future `destructive-git-guard.sh` hook is deferred from #1146.
