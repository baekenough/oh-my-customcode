# R009: Parallel Execution Rules

Two or more independent tasks MUST execute in parallel. Sequential execution of parallelizable tasks is a violation.

## Overview

Parallel execution maximizes throughput and reduces wall-clock time. Before spawning 2+ parallel agents, the Agent Teams eligibility check ([[r018]]) must be performed — Agent Teams may be required instead of raw Agent tool calls. Adaptive splitting handles stalled agents at runtime.

## Priority

**MUST** — Enforced. Sequential execution of independent tasks requires correction. PostCompact hook re-injects this rule.

## Key Requirements

**Independence criteria (MUST parallelize):**
- No shared mutable state between tasks
- No sequential dependencies
- Each completes independently

**Agent Teams gate:** Quick rule — 3+ agents OR review cycle OR 2+ issues in same batch → use Agent Teams ([[r018]]).

**Execution limits:**

| Rule | Detail |
|------|--------|
| Max instances | 5 concurrent (soft default: 4) |
| Large tasks (>3 min) | MUST split into parallel sub-tasks |
| Orchestrator | Must stay singleton, never parallelized |

**Adaptive parallel splitting:** When an agent takes 2x+ longer than peers, spawn independent follow-up tasks immediately without canceling the stalled agent.

**Display format:**
```
[1] mgr-creator:sonnet → Create Go agent
[2] lang-python-expert:sonnet → Review Python code
```

**Result aggregation:**
```
[Summary] {succeeded}/{total} tasks completed
  ✓ agent-1: success
  ✗ agent-2: failed (reason)
```

## Enforcement

Advisory PostToolUse hook + PostCompact re-injection per [[r021]].

## Relationships

- **Related rules**: [[r008]], [[r010]], [[r018]]
- **Affects**: All multi-task operations, batch workflows, research skills

## Anti-Pattern: Single Agent Giant Prompt

단일 에이전트에 5000+ 토큰 multi-domain 프롬프트를 위임하는 것은 R009 위반이다. 이 패턴은 latency timeout, user cancellation, review loop 불가능을 유발한다.

**토큰 임계치 가이드 (#1085):**

| 조건 | 권장 |
|------|------|
| < 5000 tokens, single domain | 단일 에이전트 가능 |
| ≥ 5000 tokens 또는 3+ domains | 도메인별 분할 + 병렬 에이전트 (R009) |
| ≥ 8000 tokens 또는 4+ domains | Agent Teams 사용 필수 ([[r018]]) |

```
❌ WRONG: Single agent receives massive multi-domain prompt (>5000 tokens)
   Agent(arch-documenter, prompt: <9000-token M2 plan across 7 areas>)
✓ CORRECT: Pre-decompose by domain, spawn parallel agents per area
```
