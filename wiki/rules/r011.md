# R011: Memory Integration Rules

Defines when and how agents persist knowledge across sessions using native auto-memory.

> This project uses **native auto-memory ONLY** (MEMORY.md + agent frontmatter `memory:`). The claude-mem and agentmemory MCP servers are NOT used — they were permanently removed (#1253).

## Overview

Native auto-memory (agent frontmatter `memory` field) is the sole persistence mechanism — it loads the first 200 lines of MEMORY.md into the system prompt. There are no external MCP memory backends. Memory failures must never block the main task.

## Priority

**SHOULD** — Strong recommendation. Missing memory integration reduces cross-session effectiveness but does not block operations.

## Key Requirements

**Architecture:** Native auto-memory is the single backend. No external MCP dependencies.

**Memory scope:**

| Scope | Location | Git Tracked |
|-------|----------|-------------|
| `user` | `~/.claude/agent-memory/<name>/` | No |
| `project` | `.claude/agent-memory/<name>/` | Yes |
| `local` | `.claude/agent-memory-local/<name>/` | No |

**Session-end auto-save responsibility:**

| Action | Owner |
|--------|-------|
| Session summary + native MEMORY.md update | `sys-memory-keeper` |

**Best practices:**
- Keep MEMORY.md under 200 lines
- Do not store secrets or duplicate CLAUDE.md content
- Memory write failures are non-blocking

## Subagent memory:project Source-Tree Pollution Guard

> Origin: #1335 ② — a `lang-kotlin-expert` with `memory: project`, working in a Kotlin source subdirectory, wrote memory to `mobile/.../com/baekenough/secondbrain/.claude/agent-memory/` — inside the source package. Caught and removed just before commit.

A subagent with `memory: project` that runs in a **subdirectory** can create `.claude/agent-memory/` relative to its cwd, polluting the source tree (e.g., inside a source package directory). Memory must resolve to the **project root** `.claude/`, never a nested working directory.

| Anti-pattern | Required |
|--------------|----------|
| Accept a subagent's `.claude/agent-memory/` written under a source package | Verify memory writes land at project-root `.claude/`; remove any nested `.claude/agent-memory/` before commit |

**Pre-commit check:** Before committing subagent work, scan for stray nested memory directories:

```bash
find . -path '*/src/*/.claude' -o -path '*/main/*/.claude'
```

## Mid-Session Immediate Save

Save memory IMMEDIATELY upon surprising discovery — do not defer to session end.

| Trigger | Action | Rationale |
|---------|--------|-----------|
| Repeated pattern observed (2nd time) | Save `feedback_*.md` now | Pattern will recur within session |
| Unexpected tool behavior / workaround | Save `feedback_*.md` now | Session state defense |
| Subagent false-positive detected | Save `feedback_*.md` now | Prevent repeat in same session |
| User correction / feedback | Save `feedback_*.md` now | Honor correction immediately |

**Why Immediate (not session-end):** Session-end saves lose context: by the time the session ends, multiple discoveries have compounded and nuance is lost. Immediate saves preserve the exact trigger context that makes the memory actionable.

**Anti-pattern**: "I'll batch all learnings at session end" — by then you'll have forgotten WHY each one mattered, and further violations may have occurred using the un-saved pattern.

## Session-End Retrospective Feedback (Model-Drafted)

Since `omcustom-feedback` is model-invocable (#1227), at session end the model MAY draft a retrospective feedback issue summarizing notable session friction or learnings — instead of only prompting the user to file it manually.

**Trigger conditions**: session-end detected AND notable friction/learning observed AND `omcustom-feedback` skill active.

**Key constraint**: The model NEVER auto-submits. Every draft passes through the skill's Phase 4A preview + confirmation gate before any GitHub issue is created. The user approves before publication.

| Path | Who Drafts | Who Approves |
|------|-----------|-------------|
| Manual (existing) | User | User (self-review) |
| Model-drafted (new) | Model | User via Phase 4A gate |

This complements — does not replace — the existing self-check item "prompt user to trigger omcustom-feedback". The Session-End Self-Check wording was updated to reflect that the model MAY draft instead of only prompting.

See [[omcustom-feedback]] for Phase 4A gate details.

## Enforcement

Prompt-based + Stop hook (soft block for session-end saves) per [[r021]].
