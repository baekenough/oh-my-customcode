# R019: Ontology-RAG Assisted Routing

Routing skills SHOULD enrich agent selection by calling `get_agent_for_task` from the ontology-RAG MCP server to inject contextually relevant skill suggestions.

## Overview

After static routing selects an agent, ontology-RAG provides `suggested_skills` that are prepended to the spawned agent's prompt. This is an enrichment layer — it never replaces static routing, and MCP failures must never block or delay routing.

## Priority

**SHOULD** — Strong recommendation. Ontology-RAG enrichment improves agent effectiveness but is advisory-only. MCP unavailability is silently ignored.

## Key Requirements

**Integration pattern:**
```
Static routing → agent selected
  ↓
get_agent_for_task(original_query)
  ↓
Extract suggested_skills
  ↓
Prepend to spawned agent prompt:
  "Ontology context suggests these skills may be relevant: {suggested_skills}"
```

**Failure handling:**

| Scenario | Action |
|----------|--------|
| MCP server unavailable | Skip silently, proceed with unmodified prompt |
| Empty response | Proceed with unmodified prompt |
| Response parsing error | Skip silently, log warning |

**MCP failure MUST NOT block or delay routing.**

**Scope:** Applies to all four routing skills: secretary-routing, dev-lead-routing, de-lead-routing, qa-lead-routing.

**Token impact:** ~300 tokens added per routing call — no parallelism impact.

## Enforcement

Prompt-based per [[r021]]. Advisory only.

## Relationships

- **Related rules**: [[r010]] (orchestrator calls MCP, subagent receives enriched prompt), [[r015]] (confidence display unaffected)
- **Affects**: All routing skills and spawned specialist agents
