# R008: Tool Identification Rules

Every tool call MUST be prefixed with agent and model identification so every action is traceable.

## Overview

Tool identification pairs with [[r007]] (agent identification) to create full traceability. Users see not just which agent is active but precisely which tool it is calling and on what target. The `[N]` prefix system for parallel spawns enables correlation with the Running display in Claude Code.

## Priority

**MUST** — Enforced. Missing identification prefix on tool calls requires immediate correction. PostCompact hook re-injects this rule.

## Key Requirements

**Format:**
```
[agent-name][model] → Tool: <tool-name>
[agent-name][model] → Target: <file/path/url>
```

**Model shorthand:** `opus`, `sonnet` (default), `haiku`

**Tool category verbs:**

| Category | Tools | Verb |
|----------|-------|------|
| File Read | Read, Glob, Grep | Reading / Searching |
| File Write | Write, Edit | Writing / Editing |
| Network | WebFetch | Fetching |
| Execution | Bash, Agent | Running / Spawning |

**Parallel spawn prefix rule:** When spawning 2+ agents, each Agent tool's `description` parameter MUST include a `[N]` 1-indexed prefix for Running display correlation:
```
Agent(description: "[1] Go code review", subagent_type: "lang-golang-expert")
Agent(description: "[2] Python code review", subagent_type: "lang-python-expert")
```

Single agent spawns do NOT use the `[N]` prefix.

> **v2.1.174+**: Fixed the Workflow tool's `agent()` subagents missing per-agent attribution headers. Workflow-spawned subagents now carry attribution consistent with R008. When authoring Workflow scripts, treat each `agent()` call with the same per-agent identification model as parallel Agent tool spawns — `[agent][model] → Tool:` discipline applies to Workflow fan-outs too.

## Short Response Discipline (v0.147.0)

도구 호출 prefix도 응답 길이와 무관하게 필수. 같은 턴 내 여러 도구를 호출할 때 각 호출 직전에 개별 prefix 표시:

```
[agent][model] → Tool: Read
[agent][model] → Target: file1.md
<Read call>

[agent][model] → Tool: Bash
[agent][model] → Target: gh issue list
<Bash call>
```

Reference issues: #1188 item #3, #1198 item #3.

## External-Project / Debugging Session Vigilance (#1401, #1417)

R007 헤더와 마찬가지로, R008 prefix 누락도 외부 프로젝트 디버깅·배포 세션에서 가장 자주 발생한다. R007/R008은 세트로 함께 자가 점검한다.

| 세션 유형 | R008 prefix |
|-----------|-------------|
| oh-my-customcode 작업 | 필수 |
| 외부 프로젝트 디버깅 | **동일하게 필수** |
| SSH / 배포 / 인프라 작업 | **동일하게 필수** |

외부 프로젝트 진단 세션(#1417)에서 Bash/Edit/Read/Agent 모든 호출에 `[agent][model] → Tool:` prefix가 세션 전체 누락된 재발이 관측되었다 — 도구 호출 직전 prefix 부착을 워크플로에 내재화한다.

Reference issues: #1401, #1417.
