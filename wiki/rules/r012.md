# R012: HUD Statusline Rules

Defines the two-system architecture for real-time status display: hook-based HUD events (stderr) and the persistent statusline API (stdout).

## Overview

HUD events notify users of subagent spawns and key milestones inline via hooks. The statusline API provides a persistent bottom-bar showing cost, branch, rate limits, and context usage. These are complementary channels — not redundant.

## Priority

**SHOULD** — Strong recommendation. Proper HUD/statusline integration improves session visibility, especially for long-running or parallel operations.

## Key Requirements

**Two-system architecture:**

| Aspect | HUD Events | Statusline API |
|--------|-----------|----------------|
| Channel | stderr (hooks) | stdout (dedicated statusline) |
| Location | Inline in conversation log | Persistent bar at screen bottom |
| Trigger | PreToolUse (Agent/Task matcher) | Message update cycle (~300ms) |
| Role | Event notifications | Persistent session status |

**HUD events format:**
```
─── [Spawn] {subagent_type}:{model} | {description} ───
```
Display only for multi-step/parallel/long-running operations.

**Statusline format:**
```
{Cost} | {project} | {branch} | RL:{rate_limit}% {countdown} | WL:{weekly_limit}% {countdown} | CTX:{usage}%
```

**Configuration:** `statusLine.type: "command"` + `statusLine.command: ".claude/statusline.sh"` in `.claude/settings.local.json`. Requires CC v2.1.80+. `refreshInterval` setting available in v2.1.97+.

## OTEL Observability Evolution

The HUD system integrates with OpenTelemetry telemetry progressively across CC versions:

| Version | Addition |
|---------|----------|
| v2.1.157+ | `tool_parameters` in `tool_decision` events when `OTEL_LOG_TOOL_DETAILS=1` — per-tool parameter tracking |
| v2.1.161+ | `OTEL_RESOURCE_ATTRIBUTES` emitted as labels on metric datapoints — per-dimension metric slicing |
| v2.1.172+ | `model` attribute added to `claude_code.lines_of_code.count` metric — lines-of-code telemetry can now be sliced by model |
| v2.1.174+ | `/usage` (Account & usage) dialog now shows usage attribution — cache misses, long context, subagents, per-skill/agent/plugin/MCP breakdowns over last 24h or 7d (VSCode integration). Interactive in-client cost visibility without standing up an OTEL backend. |
| v2.1.193+ | `claude_code.assistant_response` OTEL 로그 이벤트 추가 — 모델 응답 텍스트 포함. `OTEL_LOG_ASSISTANT_RESPONSES=1`이 아니면 redacted 되나, unset이면 `OTEL_LOG_USER_PROMPTS`를 따름. **보안 주의: 이미 프롬프트를 로깅하는 배포는 업그레이드 즉시 응답 내용도 수신 시작 — 프롬프트만 유지하려면 `OTEL_LOG_ASSISTANT_RESPONSES=0` 설정.** 응답 텍스트 로깅은 명시적 opt-out이 필요한 민감 항목. |

Configure via the `monitoring-setup` skill's `OTEL_RESOURCE_ATTRIBUTES` env. Each addition extends the per-dimension slicing capability, complementing the HUD parallel-spawn display.

**Rate-limit telemetry 정확도 (v2.1.196+):** 여러 병렬 요청이 사용량 한도에 도달하는 순간 rate-limit 경고가 깜빡이며 꺼지고 rate-limit telemetry가 과다 집계되던 문제를 수정했습니다. R012 관측성의 rate-limit 계측 정확도 개선입니다.

## Footer Link Badges (v2.1.176+)

The `footerLinksRegexes` setting enables regex-matched link badges rendered in the footer row, configurable via user or managed settings. The footer can now surface contextual link badges alongside the `.claude/statusline.sh` segments, extending statusline composition beyond the cost/branch/RL/WL/CTX fields. Session titles are also generated in the conversation's language from v2.1.176 (pin via the `language` setting).

## Enforcement

Prompt-based per [[r021]]. Configuration-dependent — requires settings.local.json setup.
