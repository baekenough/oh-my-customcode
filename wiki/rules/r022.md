# R022: Wiki Sync Rules

When agents, skills, rules, or guides change, corresponding wiki pages should be updated.

## Overview

The wiki is the project's compiled knowledge. Stale wikis degrade team and LLM onboarding speed. R022 establishes wiki synchronization as a recommended practice.

## Priority

**SHOULD** — advisory enforcement via CI (wiki-sync.yml). Not a hard block.

## Key Requirements

| Change Type | Wiki Action |
|-------------|-------------|
| New/modified agent | Create/update wiki/agents/ page |
| New/modified skill | Create/update wiki/skills/ page |
| New/modified rule | Create/update wiki/rules/ page |
| New/modified guide | Create/update wiki/guides/ page |

## Enforcement

CI workflow `.github/workflows/wiki-sync.yml` checks for missing wiki pages on every PR.
