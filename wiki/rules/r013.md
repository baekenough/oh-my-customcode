# R013: Ecomode Rules

Defines automatic token efficiency mode for batch, parallel, and high-context operations — compressing both output and input context.

## Overview

Ecomode activates automatically when 4+ parallel tasks are running, batch operations are detected, context usage exceeds 80% (or lower task-type-specific thresholds), or the user explicitly enables it. Both output compression and input context pruning are active under ecomode.

## Priority

**SHOULD** — Strong recommendation. Ecomode prevents context overflow and reduces token costs in large operations. Override with "ecomode off" or "verbose mode".

## Key Requirements

**Auto-activation triggers:**
- 4+ parallel tasks
- Batch operations
- Context usage > threshold (task-type-specific)
- Explicit "ecomode on"

**Task-type context thresholds:**

| Task Type | Trigger |
|-----------|---------|
| Research | 40% |
| Implementation | 50% |
| Review | 60% |
| Management | 70% |
| General | 80% |

**Aggregation format:**
```
[Batch Complete] {n}/{total}
├── {agent}: ✓/✗/⚠ {summary}
```

**Input context pruning strategies:**

| Strategy | When |
|----------|------|
| Retain | Directly relevant code/docs |
| Summarize | Background context, prior hop results |
| Drop | Search noise, duplicates, already-reflected info |

Track pruning: `[Pruned] {N} chunks → {M} retained, {K} summarized, {J} dropped`

## Enforcement

Prompt-based + `context-budget-advisor.sh` hook (advisory warnings) per [[r021]].
