# R006: Agent Design Rules

Defines the file format, required frontmatter, optional fields, naming conventions, and separation-of-concerns architecture for all agents and skills.

## Overview

Agents live in `.claude/agents/{name}.md`. Skills live in `.claude/skills/{name}/SKILL.md`. Guides in `guides/`. These three locations have distinct purposes and must not be conflated. The agent frontmatter defines identity, tools, model, and behavior. The body describes role and workflow — not detailed instructions (those belong in skills).

## Priority

**MUST** — Agent files with invalid frontmatter or wrong structure cause routing failures and R017 verification failures.

## Key Requirements

**Required frontmatter fields:** `name`, `description`, `model`, `tools`

**Model aliases:**

| Alias | Full ID | Use Case |
|-------|---------|----------|
| `haiku` | claude-haiku-4-5 | Fast, cheap tasks |
| `sonnet` | claude-sonnet-4-6 | General tasks (default) |
| `opus` | claude-opus-4-6 | Complex reasoning |
| `opusplan` | claude-opus-4-6 + plan mode | Architecture planning |

**Separation of concerns:**

| Location | Purpose | Contains |
|----------|---------|----------|
| `.claude/agents/` | WHAT the agent does | Role, capabilities, workflow |
| `.claude/skills/` | HOW to do tasks | Instructions, scripts, rules |
| `guides/` | Reference docs | Best practices, tutorials |

**Skill scope:**

| Scope | Deployed via init? |
|-------|--------------------|
| `core` | Yes |
| `harness` | Yes |
| `package` | No |

**Protected paths:** `.claude/agents/*.md`, `.claude/skills/*/SKILL.md`, `guides/*/` (new dirs) — creation MUST go through `mgr-creator` per [[r010]].

## Enforcement

Prompt-based + mgr-sauron R017 verification. mgr-creator enforces frontmatter validation on creation.
