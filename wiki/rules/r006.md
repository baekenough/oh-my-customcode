# R006: Agent Design Rules

Defines the file format, required frontmatter, optional fields, naming conventions, and separation-of-concerns architecture for all agents and skills.

## Overview

Agents live in `.claude/agents/{name}.md`. Skills live in `.claude/skills/{name}/SKILL.md`. Guides in `guides/`. These three locations have distinct purposes and must not be conflated. The agent frontmatter defines identity, tools, model, and behavior. The body describes role and workflow ŌĆö not detailed instructions (those belong in skills).

## Priority

**MUST** ŌĆö Agent files with invalid frontmatter or wrong structure cause routing failures and R017 verification failures.

## Key Requirements

**Required frontmatter fields:** `name`, `description`, `model`, `tools`

**Model aliases:**

| Alias | Full ID | Use Case |
|-------|---------|----------|
| `haiku` | claude-haiku-4-5 | Fast, cheap tasks |
| `sonnet` | claude-sonnet-4-6 | General tasks (default) |
| `opus` | claude-opus-4-6 | Complex reasoning |
| `opusplan` | claude-opus-4-6 + plan mode | Architecture planning |

**Separation of concerns:**

| Location | Purpose | Contains |
|----------|---------|----------|
| `.claude/agents/` | WHAT the agent does | Role, capabilities, workflow |
| `.claude/skills/` | HOW to do tasks | Instructions, scripts, rules |
| `guides/` | Reference docs | Best practices, tutorials |

**Skill scope:**

| Scope | Deployed via init? |
|-------|--------------------|
| `core` | Yes |
| `harness` | Yes |
| `package` | No |

**Protected paths:** `.claude/agents/*.md`, `.claude/skills/*/SKILL.md`, `guides/*/` (new dirs) ŌĆö creation MUST go through `mgr-creator` per [[r010]].

## Artifact Output Convention

Skills persist output to `.claude/outputs/sessions/{YYYY-MM-DD}/{skill-name}-{HHmmss}.md`. Opt-in, git-untracked. Final subagent writes per [[r010]].

## Sensitive Path Handling

`.claude/` is treated as a sensitive directory by CC. The sensitive-path check runs **above** `bypassPermissions` and explicit allow rules ŌĆö prompts may appear even with `Write(.claude/**)` in settings.

| Path | Tool | Allow rule | Result |
|------|------|-----------|--------|
| `.claude/**` | Bash (`cp`, `mkdir`, `rm`) | `Bash(*)` allowed | Prompt (overrides allow) |
| `.claude/**` | Write, Edit | `Write(.claude/**)` allowed | Prompt (overrides allow) |
| `templates/.claude/**` | Write, Edit | `Write(templates/.claude/**)` allowed | Prompt (confirmed CC v2.1.116+; see #960, #961, #981) |
| `.claude/outputs/**` | Any | ŌĆö | Allowed (artifact convention) |

**Recommended practice:**
1. Prefer `Write`/`Edit` over `Bash(cp)`/`Bash(mkdir)` for `.claude/` paths ŌĆö better auditability
2. Add defensive allow rules (`Write(.claude/**)`, `Write(templates/.claude/**)`, etc.) ŌĆö `.claude/**` glob does not cover `templates/.claude/**`; separate entries required

## Enforcement

Prompt-based + mgr-sauron R017 verification. mgr-creator enforces frontmatter validation on creation.
