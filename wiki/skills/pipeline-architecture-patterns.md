# Pipeline Architecture Patterns

Data pipeline architecture patterns for ETL/ELT design, orchestration, and data quality frameworks.

## Overview

Reference patterns for data pipeline design: ETL vs ELT, Lambda vs Kappa vs Medallion architectures, DAG-based and event-driven orchestration, data contracts, validation frameworks (Great Expectations, dbt tests, Soda), data lineage, and idempotency patterns. Used by `de-pipeline-expert` and DE specialists when designing pipeline architectures.
